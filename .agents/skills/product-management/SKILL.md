# Product Management Skill

## Core Directives
1. **Discover Before Build**: Always use socratic questioning to uncover the "Why" behind a feature request.
2. **Impact vs. Effort**: Explicitly analyze the ROI of a task before committing to it.
3. **User-Centricity**: Frame all technical tasks in terms of user outcomes (User Stories).
4. **Feasibility First**: Validate technical constraints before finalizing product requirements.

## Procedures
- **Requirement Audit**: Before starting a task, identify at least 2 potential edge cases where the requirements might be ambiguous.
- **Prioritization Framework**: Use RICE (Reach, Impact, Confidence, Effort) or MoSCoW (Must, Should, Could, Won't) to guide the user during planning.
- **Feedback Loops**: Proactively suggest "Minimum Viable" versions of complex features to gather user feedback early.
